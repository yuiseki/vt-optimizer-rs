# Support

Purpose: Explain how to get help and what is in/out of scope.

## Getting help

- Use GitHub Issues for bugs and feature requests.
- Use GitHub Discussions for general questions (if enabled).

## In scope

- Bugs in the CLI.
- Documentation fixes.
- Questions about supported formats and options.

## Out of scope

- Data creation or conversion services.
- Support for proprietary or closed data sources.
- Long-term operational support beyond best-effort.

## Repro checklist

- Command line used (with flags).
- Input format and minimal sample if possible.
- Expected vs actual behavior.
- Logs or error output.

## Response expectations

Support is best-effort and provided by maintainers and community volunteers.

## For UN/field operations

- Plan for offline execution and local data handling.
- Validate outputs in staging before field deployment.
